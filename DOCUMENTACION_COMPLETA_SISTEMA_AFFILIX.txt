1. RESUMEN DEL SISTEMA
===============================================================================

AFFILIX es una plataforma de comercio mediante afiliación. Su función principal
es reunir productos procedentes de proveedores y redes de afiliados, publicarlos
en una tienda pública y dirigir al comprador al enlace afiliado correcto.

El sistema está dividido en cuatro áreas:

1. Tienda pública:
   Muestra productos, ofertas, categorías, buscador, fichas y enlaces de compra.

2. Panel principal de administración:
   Permite gestionar productos, proveedores, afiliados, inteligencia artificial,
   automatizaciones, analítica, marketing, Meta Ads y configuración.

3. Panel de afiliados AFFILIX Partners:
   Cada afiliado puede seleccionar productos autorizados, publicarlos en su
   escaparate, revisar clics, ventas y comisiones, configurar su web y preparar
   promociones.

4. Backend y base de datos:
   Guarda productos, conexiones, clics, comisiones, afiliados, campañas,
   notificaciones y registros de automatización.

AFFILIX no es un proveedor ni una pasarela de pago. Cuando un comprador pulsa
"Comprar", AFFILIX registra el clic y lo envía al proveedor mediante el enlace
afiliado. El pago, envío, devolución y entrega corresponden al proveedor.

El principio central es:

PROVEEDOR CONECTADO -> PRODUCTO REAL -> ENLACE AFILIADO REAL ->
PUBLICACIÓN EN AFFILIX -> CLIC REGISTRADO -> REDIRECCIÓN AL PROVEEDOR

2. TECNOLOGÍA UTILIZADA
===============================================================================

Frontend y servidor:
- Next.js 16.2.6 con App Router.
- React 18.
- TypeScript.
- Tailwind CSS.
- Rutas dinámicas y endpoints dentro del mismo proyecto.

Base de datos:
- Supabase/PostgreSQL.
- Acceso administrativo mediante la REST API de Supabase.
- Políticas RLS definidas en las migraciones.

Seguridad y sesiones:
- Cookies HTTP-only.
- Sesión administrativa independiente.
- Sesión de afiliado firmada con HMAC-SHA256.
- Contraseñas de afiliados cifradas con bcrypt, coste 12.
- Cifrado de credenciales externas antes de guardarlas.
- Control de origen, cabeceras de seguridad, detección básica de bots y límite
  de peticiones para endpoints API.

Servicios externos:
- Vercel para alojamiento y cron jobs.
- Supabase para datos.
- Upstash Redis, cuando está configurado, para caché y limitación distribuida.
- OpenAI, Anthropic, Groq, Mistral u Ollama para funciones de texto.
- OpenAI o endpoint compatible personalizado para imágenes.
- Meta Graph API v24.0 para Facebook, Instagram y Meta Ads.
- Pinterest API v5.
- X/Twitter API v2.
- Resend para correo, cuando se configura su API.

4. TIENDA PÚBLICA
===============================================================================

Rutas principales:
- / o /store: portada de la tienda.
- /productos: catálogo completo.
- /productos/[slug]: ficha de producto.
- /ofertas: productos destacados y ofertas.
- /go/[slug]: tracking y redirección al proveedor.

Contenido de la portada:
- Cabecera con logotipo.
- Buscador.
- Navegación por departamentos.
- Ofertas visuales destacadas.
- Hero con oferta destacada cuando hay producto disponible.
- Compra por proveedor.
- Ofertas de la semana.
- Catálogos de productos nuevos.
- Carril de productos nuevos.
- Carril de ofertas destacadas.
- Carril de productos populares.
- Filtros por categoría, plataforma y orden.
- Pie de página con información comercial.

Buscador público:
- Busca productos por texto.
- Ofrece sugerencias mientras el usuario escribe.
- Permite pulsar una sugerencia sin completar toda la palabra.
- Busca por título, categoría y términos disponibles en el catálogo.
- Conserva filtros mediante parámetros de URL.

Catálogo:
- Solo debe mostrar productos activos.
- Organiza productos por fecha, popularidad, categoría y plataforma.
- Muestra imagen, título, precio, proveedor y llamada a la acción.
- Si no existen productos activos, muestra un estado vacío real.
- No se deben inventar productos para rellenar la tienda.

Ficha de producto:
- Imagen principal.
- Título generado o título original.
- Precio, precio anterior y descuento cuando existen.
- Categoría y proveedor.
- Descripción.
- Datos rápidos.
- Productos relacionados.
- Botón de compra.

Flujo de compra:
1. El cliente abre una ficha.
2. Pulsa el botón de compra.
3. El enlace pasa por /go/[slug].
4. AFFILIX registra el clic.
5. AFFILIX redirige al affiliate_url real del proveedor.

La comisión solo puede atribuirse correctamente si affiliate_url contiene el
tracking real de la cuenta conectada. Un enlace normal sin identificador de
afiliado puede generar una venta sin comisión para AFFILIX.

7. PANEL "PROVEEDORES"
===============================================================================

Ruta:
- /dashboard/providers

Función:
- Mostrar y organizar todas las plataformas disponibles.
- Separarlas por acceso gratuito, de pago o premium.
- Mostrar su web oficial y requisitos de conexión.
- Guardar credenciales.
- Probar la conexión.
- Importar un producto mediante enlace cuando corresponde.
- Mostrar tendencias o productos obtenidos cuando una API ofrece catálogo.

Estados posibles:
- No configurado: no hay datos guardados.
- Guardado: existen datos, pero no han sido probados.
- Conectado: la prueba real terminó correctamente.
- Error: la API rechazó la conexión o faltan datos.
- Publicación por enlace: la plataforma no ofrece catálogo completo, pero se
  puede publicar un enlace afiliado aprobado.

Importante:
- "Conectado" significa que las credenciales o el modo de enlace se validaron.
- No significa que el proveedor entregue automáticamente todo su marketplace.
- Algunas redes solo muestran productos aprobados para esa cuenta.
- Algunas APIs solo muestran productos propios del vendedor.
- Captcha, 2FA y aprobación humana no pueden sustituirse con una API.

9. CONEXIONES DE PROVEEDORES IMPLEMENTADAS
===============================================================================

Con prueba de API o conector específico:

Amazon Associates:
- Requiere Access Key, Secret Key y Associate Tag.
- Usa Product Advertising API.
- Puede buscar productos, novedades y superventas.
- Amazon puede exigir ventas calificadas para mantener activa la API.

Amazon Seller Central:
- Requiere Seller ID, Marketplace ID, LWA Client ID, LWA Client Secret y
  Refresh Token.
- La validación necesita autorización SP-API real.

ClickBank:
- Requiere API Key y Nickname.
- Puede consultar marketplace.
- Puede filtrar productos por rentabilidad y gravity.

CJ Affiliate:
- Requiere API Key y Website ID.
- Puede usar Product Search API.

Hotmart:
- Requiere Client ID y Client Secret o Basic Token.
- Valida credenciales.
- Los productos afiliados aprobados pueden importarse mediante HotLink.

Gumroad:
- Requiere Access Token.
- Lee productos publicados por esa cuenta.
- Si la cuenta tiene cero productos propios publicados, la API devuelve cero.

WarriorPlus:
- Requiere API Key.
- Consulta productos, ofertas y solicitudes de afiliado.
- Si no existen productos propios ni solicitudes aprobadas, devuelve cero.

Digistore24:
- Requiere una API Key válida.
- La prueba usa X-DS-API-KEY.
- Para productos afiliados se permite publicación mediante enlace aprobado.

JVZoo:
- Requiere API Key.
- La prueba usa autenticación Basic con API_KEY:x.
- La API valida la cuenta, pero no entrega todo el marketplace como catálogo.
- Los productos se publican con enlaces aprobados por el vendedor.

Payhip:
- Guarda y prueba la conexión disponible.
- La API pública no ofrece un feed completo para este uso.
- Se publica mediante enlace real de producto y foto pública.

systeme.io:
- Requiere Public API Key.
- Puede publicar funnels o productos mediante enlace real.

Plataformas sin catálogo automático específico:
- Las credenciales pueden guardarse y el proveedor puede activarse en modo
  publicación por enlace.
- AFFILIX no debe presentar ese modo como un feed API completo.

10. PANEL "SOLICITUDES AFILIADO"
===============================================================================

Ruta:
- /dashboard/affiliate-scout

Objetivo:
- Simplificar el trabajo con proveedores que exigen aprobación.

Flujo mostrado:
1. Entrar al proveedor.
2. Pedir aprobación.
3. Pegar el enlace afiliado.
4. Publicar el producto.

El panel ofrece:
- Botón para abrir el login oficial.
- Botón para abrir el marketplace.
- Botón para guardar la API o datos.
- Formulario para pegar el enlace afiliado aprobado.
- Precio y porcentaje de comisión opcionales.
- Nichos recomendados para iniciar búsquedas.
- Separación entre proveedores conectados y pendientes.

Por qué no guarda correos y contraseñas de proveedores:
- Compartir contraseñas de terceros dentro de AFFILIX es un riesgo.
- Los proveedores pueden activar captcha, 2FA y controles de dispositivo.
- Un robot de login se rompe cuando cambia el diseño de la web.
- Puede incumplir las condiciones del proveedor.

La solución estable consiste en:
- Abrir el login oficial.
- Dejar que el navegador del propietario recuerde la contraseña.
- Guardar en AFFILIX solo API keys, tokens o enlaces aprobados.
- Automatizar la importación y publicación después de obtener autorización.

12. IMPORTACIÓN Y PUBLICACIÓN DE PRODUCTOS
===============================================================================

Endpoints principales:
- POST /api/products/import
- POST /api/products/import-link
- POST /api/products/publish
- POST /api/products/image
- POST /api/products/fetch-provider-image
- GET/POST /api/product-image/[id]

Importación desde API:
1. El proveedor debe estar conectado.
2. El buscador obtiene un producto externo.
3. El administrador confirma la importación.
4. AFFILIX guarda producto, imagen, enlace y plataforma.

Importación por enlace:
1. El administrador pega el enlace del producto.
2. El sistema verifica que la plataforma esté activa.
3. Descarga la página pública si es accesible.
4. Intenta extraer título, descripción e imagen Open Graph.
5. Guarda el enlace exacto como affiliate_url.
6. Publica el producto.

Condición esencial:
- El enlace pegado debe ser el enlace afiliado que pertenece a la cuenta.
- AFFILIX no puede convertir un enlace normal en afiliado si el proveedor no
  ofrece identificador, API o generador de tracking.

Fotografías:
- Fuente del proveedor: se usa la imagen oficial de la ficha.
- Manual: el administrador sube una nueva imagen.
- IA: el sistema genera una imagen cuando hay API configurada.
- La imagen debe ser pública y accesible por Meta, Instagram y Pinterest.

27. SEGURIDAD
===============================================================================

Medidas existentes:
- Cookies HTTP-only.
- Cookies Secure en producción.
- SameSite Lax.
- Contraseñas de afiliados con bcrypt.
- Sesiones de afiliado firmadas.
- Credenciales externas cifradas.
- Filtrado de orígenes.
- Rate limit de API.
- Detección básica de robots.
- Cabeceras de seguridad.
- Separación de sesión administrativa y afiliada.
- RLS en tablas.
- Cron protegido por secreto.
- Webhooks preparados para firma.

Puntos que deben conservarse al modificar:
- Nunca exponer SUPABASE_SERVICE_ROLE_KEY al navegador.
- Nunca mostrar tokens completos en HTML.
- Nunca guardar contraseñas de proveedores en texto plano.
- Nunca desactivar la validación de proveedor antes de publicar.
- Nunca permitir que un afiliado consulte datos de otro.
- Nunca confiar en precios, comisiones o ventas enviados solo por el navegador.


===============================================================================
28. LIMITACIONES REALES DEL SISTEMA
===============================================================================

1. Una API conectada puede devolver cero productos.
   Esto ocurre si la cuenta no tiene productos propios, ofertas o aprobaciones.

2. Conectar una cuenta no concede acceso a todo el marketplace.
   JVZoo, WarriorPlus, Impact y otras redes aplican aprobaciones del vendedor.

3. AFFILIX no puede garantizar una comisión con un enlace normal.
   Debe usarse el enlace afiliado aprobado.

4. Una imagen no siempre puede copiarse automáticamente.
   Algunas webs bloquean descargas, usan JavaScript o prohíben hotlinking.

5. Las ventas no se conocen solo por los clics.
   Se necesita webhook, API de conversiones o confirmación del proveedor.

6. Un robot de correo y contraseña no es una conexión estable.
   Captcha, 2FA y términos del proveedor pueden bloquearlo.

7. Meta Ads necesita autorización y método de pago.
   AFFILIX prepara y controla campañas, pero Meta decide permisos y cobros.

8. La IA necesita una API válida.
   Sin API, no puede generar contenido ni imágenes externas.

9. Los cron jobs solo ejecutan correctamente si Vercel tiene las variables y
   Supabase responde.

10. La base de datos en vivo no pudo consultarse durante esta documentación
    porque el dominio del proyecto Supabase no resolvió desde el entorno local.
    Por ello este archivo documenta el código y el despliegue verificado, pero
    no afirma cantidades actuales de productos, clics o cuentas conectadas.

29. FLUJO OPERATIVO RECOMENDADO
===============================================================================

Para dejar un proveedor produciendo:
1. Abrir /dashboard/providers.
2. Registrarse en la web oficial.
3. Solicitar acceso de afiliado si es necesario.
4. Crear API key o token cuando exista.
5. Guardar los datos.
6. Pulsar "Probar conexión real".
7. Confirmar estado conectado.
8. Buscar productos por API o abrir /dashboard/affiliate-scout.
9. Solicitar aprobación del producto.
10. Copiar el enlace afiliado aprobado.
11. Importarlo.
12. Revisar título, precio, imagen y comisión.
13. Abrir la ficha pública.
14. Probar el botón de compra.

Para publicar en redes:
1. Conectar cuentas sociales.
2. Probar sus tokens.
3. Seleccionar un producto con imagen pública.
4. Generar contenido.
5. Revisar el texto.
6. Publicar.
7. Comprobar publish_log.

Para Meta Ads:
1. Configurar cuenta publicitaria y página.
2. Probar Meta.
3. Elegir presupuesto.
4. Crear campaña.
5. Revisarla en Meta Ads Manager.
6. Activarla cuando esté correcta.
7. Sincronizar resultados.

Para crear un afiliado:
1. Abrir /dashboard/affiliates.
2. Crear usuario y contraseña inicial.
3. Definir marca y reparto.
4. Probar /affiliate/login.
5. Seleccionar productos permitidos.
6. Publicarlos en su escaparate.
7. Probar su URL /a/[storeSlug].

- Tienda pública.
- Catálogo y fichas de producto.
- Tracking de clics.
- Panel administrativo.
- Gestión de proveedores.
- Importación por API y enlace.
- Imágenes manuales, del proveedor o IA.

- Analítica y comisiones.
- Panel completo de afiliados.

- Notificaciones.
- Despliegue en Vercel.
- Base de datos Supabase.

La parte que determina que el sistema genere ingresos no es solo el diseño o la
conexión técnica. Cada producto necesita:

1. Proveedor realmente conectado.
2. Aprobación de afiliado cuando sea obligatoria.
3. Enlace afiliado que pertenezca a AFFILIX.
4. Imagen y ficha reales.
5. Producto activo en la tienda.
6. Tracking de conversión disponible para confirmar ventas.

Este documento representa la arquitectura y las funciones encontradas en el
código del proyecto a fecha 4 de julio de 2026. Debe actualizarse después de
cambios importantes en paneles, proveedores, base de datos o despliegue.
